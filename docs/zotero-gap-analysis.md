# Lume 对标 Zotero 的功能差距分析

> 首次版本：2026-03-07 | **最后更新：2026-03-16**
> 目的：基于当前仓库现状，梳理 Lume 作为"学术论文阅读软件"要对标 Zotero 仍缺少的关键能力。
> 说明：本文档只做产品与功能分析，不包含代码实现。
> **标注说明：** ✅ 已完成 | 🔧 部分实现 | ❌ 尚未开始

---

## 一、当前版本已经具备的能力

### 原有基础能力（v0.1 骨架）

**文献库基础**
- 本地论文库目录（`My Library`）
- 文件夹 / 子文件夹管理
- PDF 导入、删除、重命名
- 根目录查看全部论文，子目录查看当前目录论文
- 左侧文件夹树 + 中间论文列表

**PDF 阅读**
- 多标签切换、页面渲染与懒加载
- 页码跳转、放大 / 缩小、适应宽度 / 适应高度
- 文本层加载与文本选择

**基础批注**
- 手写绘制、荧光笔标注、文本批注
- 批注本地保存（JSON sidecar）

---

### ✅ 新增完成能力（2026-03-07 ～ 2026-03-09）

#### ✅ 完整元数据编辑面板（对应 P0 #2）
- 所有 bibliographic 字段均可内联编辑：标题、作者、年份、期刊、卷/期/页、出版社、DOI、arXiv、ISBN、URL、语言、摘要
- MetaPanel 切换 View / Edit 双模式，保存后本地即时刷新
- 自动从 Crossref / arXiv API 联网抓取并补全元数据（DOI → Crossref，arXiv ID → arXiv，互为回退）

#### ✅ 全局检索（对应 P0 #3）
- 全库 SQL 搜索，不受当前文件夹限制
- 字段过滤 pill：All Fields / Title / Authors / Year / DOI / arXiv
- 年份精确筛选输入框
- 搜索结果关键词高亮（amber mark）
- 搜索结果显示所在文件夹路径
- 标签过滤与关键词搜索可叠加（AND 逻辑）

#### ✅ 完整标签系统工作流（对应 P0 #4）
- 左侧侧边栏独立 Tags 区域，展示全部标签及引用计数
- 点击标签快速过滤文献列表；再次点击取消
- 文献列表 Row 2 展示当前激活标签 chip，支持一键清除
- 右键标签打开颜色选择器（8 色预设调色板）
- MetaPanel 内联 pill 编辑器：Enter / 逗号添加，Backspace 删除，× 移除单个标签
- 标签彩色圆点在 View 模式同步显示
- 数据持久化至 SQLite：`item_tags` + `tag_colors` 两张表

#### ✅ 引用与参考文献生成（对应 P0 #6）
- MetaPanel Cite 区：6 种格式 pill 切换，实时预览，一键复制
- LibraryView 新增 Export 按钮：批量导出当前可见列表全部文献
- 支持 7 种格式：**APA 7th / MLA 9th / Chicago 17th / GB/T 7714-2015 / BibTeX / RIS / CSL JSON**
- 下载生成对应扩展名文件（`.bib` / `.ris` / `.json` / `.txt`）
- 全部格式由 Rust 原生生成，无外部 CSL 运行时依赖

---

### ✅ 新增完成能力（2026-03-09 ～ 2026-03-10）

#### ✅ 完整文献条目模型（对应 P0 #2）
- 支持多种条目类型（期刊论文 / 会议论文 / 书籍等），后端结构解耦
- 支持一个条目挂载多个异构附件（PDF、纯元数据条目无需附件等设计基础已奠定）
- 数据库结构由 PDF 驱动重构为 Item 驱动（`items` + `attachments`），支持完整的增删改查生态

#### ✅ PDF 内文本搜索（对应 P1 #4）
- 浮动搜索框（Ctrl+F 唤起）
- 关键词高亮及多结果跳转导航
- 提升了 PDF 阅读器的文献内信息的索引和使用体验

#### ✅ 论文级笔记系统（对应 P0 #1）
- MetaPanel 增加 Notes 工作区，集成 Markdown 编辑器
- 笔记内容纳入 SQLite 独立存储，并支持全局检索
- 引入追加已有批注至笔记的能力（整合碎片知识），补齐学术阅读到知识卡片的沉淀链路

#### ✅ 批注管理工作流 v1（对应 P0 #3）
- MetaPanel 内新增统一批注列表，按页码与页内位置展示文字 / 高亮 / 手写批注
- 支持类型筛选、文字批注关键词搜索、结果计数与点击跳转
- 支持将批注导出为 Markdown，并与“提取批注到笔记”共用同一套整理格式

## 二、仍然缺少的能力

| 优先级 | 功能项 | 状态 | 说明 |
|--------|--------|------|------|
| P0 | 论文级笔记系统 | ✅ | 围绕每篇论文的独立 Markdown 笔记 |
| P0 | 文献条目类型 & 多附件 | ✅ | 条目模型重构并支持多种类型、多附件挂载 |
| P0 | 批注管理视图 | ✅ | 统一批注列表、筛选、搜索、跳转与 Markdown 导出 |
| P1 | PDF 内文本搜索 | ✅ | Ctrl+F 关键词搜索 + 高亮跳转 |
| P1 | 文献导入生态 | 🔧 | 已支持导入 PDF 时自动解析并联网补全元数据，仍缺 BibTeX/RIS 导入、拖入入口与批量流程 |
| P1 | 重复文献识别与合并 | ❌ | DOI/哈希去重 + 合并流程 |
| P1 | Smart Collections | ❌ | 按条件自动聚合，脱离文件夹思维 |
| P1 | 元数据多源补全 & 冲突 UI | 🔧 | 当前仅 Crossref/arXiv，无批量/冲突确认 |
| P2 | BibTeX / RIS 文件导入 | ❌ | 承接 Zotero 用户迁移的关键 |
| P2 | 多设备同步 | ❌ | iCloud Drive 或自定义路径同步 |
| P2 | 写作集成 | ❌ | 全局 Cite Picker、Obsidian/Logseq 导出 |

| P3 | OCR 扫描件识别 | ❌ | Tesseract 集成 |

## 三、P0：下一阶段必须补齐的核心能力

### 1. 论文级笔记系统（已完成 ✅）
**最新进展（2026-03-10）：** MetaPanel 已新增 Notes 标签页，内嵌 Markdown 编辑器。笔记数据持久化至 SQLite（`notes` 表），内容已纳入全局搜索。
**功能亮点：** 学术阅读的终点已可沉淀为"知识卡片"，打通了阅读工具向研究工作台的能力。

---

### 2. 文献条目模型（已完成 ✅）
**最新进展（2026-03-10）：** 已基于 `items` 和 `attachments` 的数据库结构重构，脱离了单纯的 PDF 为中心的架构。支持多种条目类型，以及一个条目挂载多个记录的基础设计。

---

### 3. 批注管理视图（已完成 ✅）
**最新进展（2026-03-16）：** MetaPanel 已支持统一批注列表，按页码与页内纵向位置展示文字 / 高亮 / 手写批注；支持类型筛选、文字搜索、点击跳转与 Markdown 导出。
**功能亮点：** 批注已从"只能写"升级为"可回顾、可检索、可导出"，并与笔记系统共享同一套 Markdown 整理链路。

---

## 四、P1：强烈建议补齐的高价值能力

### 4. 阅读器基础能力仍偏弱 🔧
**当前现状：** 渲染、翻页、批注可用，但缺少现代 PDF 阅读器的标准功能。

**当前已有：** 渲染、懒加载、翻页、缩放、文本层、多种批注工具、**PDF 内文本搜索（Ctrl+F 高亮跳转）✅**

**仍缺少：**
- 目录 / Outline / 书签侧边栏
- 缩略图导航侧栏
- 快捷键体系（← → 翻页、Ctrl+F 搜索、Ctrl+= 缩放等）
- 历史阅读位置记忆（关闭后重新打开恢复位置）
- 最近打开文献列表
- 橡皮擦 / 撤销 / 重做批注操作

**为什么重要：**
阅读器体验是用户留存的底盘，这些是用户每天使用都会感知到的摩擦点。

---

### 5. 文献导入方式太少 🔧
**当前现状：** 已支持导入 PDF 时自动解析本地元数据，并尽可能通过 Crossref / arXiv 联网补全；但主入口仍是本地文件选择器，外部格式导入与批量工作流仍缺失。

**当前已有：** 本地 PDF 导入、导入时自动解析并联网补全元数据、DOI/arXiv 联网抓取（手动补全入口仍保留）

**仍缺少：**
- "通过 DOI / arXiv ID 新建条目"入口（先建 item，再关联 PDF 或不关联）
- 拖入 PDF 后自动联网识别并填写元数据（拖入入口本身仍未提供）
- BibTeX / RIS / CSL JSON 文件**导入**（批量建库）
- 批量导入时的进度反馈与错误报告
- 文件夹监听（指定目录，新增 PDF 自动入库）

**为什么重要：**
用户的真实习惯是"随手把论文丢进来"，而不是"逐个填写元数据"。自动识别可大幅降低入库摩擦。

---

### 6. 重复文献识别与合并缺失 ❌
**当前现状：** 无任何去重机制。

**缺少内容：**
- 导入时检测相同 DOI / 文件哈希的重复项
- 重复项视图（高亮显示冲突字段）
- 合并：保留最优 PDF + 最优元数据
- 去重报告

---

### 7. 文献组织方式仍是纯文件夹思维 ❌
**当前现状：** 完全依赖物理目录树，没有跨文件夹的逻辑聚合能力。

**缺少内容：**
- Smart Collections / Saved Searches（按条件自动聚合）
- 系统虚拟视图：未读 / 最近添加 / 最近打开 / 已加星标
- 同一条目出现在多个逻辑集合中（不复制文件）
- 按年份 / 作者 / 期刊的虚拟分组视图

---

### 8. 元数据来源与校正能力不足 🔧
**当前现状：** 已支持 Crossref + arXiv 双源，但无多源回退 UI 和冲突确认流程。

**仍缺少：**
- Semantic Scholar / OpenAlex / PMID / ISBN 等额外来源
- 批量补全元数据（对整个文件夹批量触发联网抓取）
- 元数据冲突对比 UI（显示"本地 vs API 返回"差异，让用户选择）
- 缺字段检测（在文献列表中标记元数据不完整的条目）

---

## 五、P2：做强产品竞争力的重要能力

### 9. BibTeX / RIS 文件导入 ❌
**当前现状：** 只能导出，无法从外部引用管理器批量导入。

**缺少内容：**
- 解析 `.bib` 文件，批量创建 items（含元数据）
- 解析 `.ris` 文件
- 从 Zotero 导出的 Better BibTeX 格式兼容

**为什么重要：**
这是 Lume 能否承接 Zotero 用户迁移的关键能力。

---

### 10. 同步能力缺失 ❌
**当前现状：** 完全单机本地应用。

**缺少内容：**
- iCloud Drive / 自定义路径数据库同步
- 批注和笔记跨设备同步
- 冲突解决策略

---

### 11. 写作集成能力缺失 ❌

**缺少内容：**
- 全局 Cite Picker（快捷键唤起，搜索并复制引用到写作工具）
- Obsidian / Logseq 格式笔记导出
- Overleaf / LaTeX 工作流（BibTeX 导出已部分覆盖）

---

### 12. 团队协作能力缺失 ❌
- 团队共享文献库
- 共享标签体系
- 评论与讨论流

---

## 六、P3：体验层值得补齐的能力

### 13. 基础桌面体验
- 阅读器 / 文献列表快捷键体系（全局与局部分层）
- 全局快捷键注册（系统级，非 PDF 内）
- 窗口 / 标签状态恢复（重启后恢复上次会话）
- 收藏 / 置顶文献
- 回收站（软删除 + 批量恢复）
- 批量操作：多选后批量打标签 / 批量移动 / 批量导出

### 14. 性能与大库体验
- 上万篇论文下的列表虚拟滚动（当前为全量渲染）
- 后台全文索引（SQLite FTS5）
- 元数据异步刷新队列（不阻塞 UI）
- 增量目录扫描（只处理新增/变更文件）

### 15. 扫描件与 OCR
- OCR 识别扫描版 PDF（Tesseract 集成）
- OCR 文本纳入全文搜索

## 七、人性化体验角度仍缺少的细节能力

以下功能不一定决定“能不能用”，但会明显影响用户是否愿意长期使用、是否觉得产品“顺手”。这些细节建议作为体验层补充项单独推进。

### 16. 空状态与首次引导不足
**当前问题：** 新用户进入应用后，需要自行猜测导入、建文件夹、开始阅读的路径。

**建议补充：**
- 首次启动欢迎页：告诉用户“先导入 PDF / 新建文件夹 / 从 DOI 建条目”
- 空文件夹空状态提示：显示推荐操作按钮，而不是只展示空白区域
- 关键区域微文案：例如 Tags、Notes、Annotations 区域给出一句用途说明
- 首次使用批注 / 搜索 / 标签时给出一次性轻提示

**为什么重要：**
降低学习成本，让第一次打开应用的用户不需要“自己摸索产品逻辑”。

---

### 17. 删除 / 移动 / 重命名后的反馈仍偏弱
**当前问题：** 用户完成操作后，容易不确定“到底成功没、改到了哪里”。

**建议补充：**
- 统一 toast 反馈：已删除、已移动到某文件夹、已重命名
- 重要操作增加 Undo（撤销）入口，特别是删除、移动、批量修改标签
- 删除前显示更明确的影响范围，例如“将删除该文件夹及其下 N 篇论文”
- 导入失败 / 移动失败给出可理解原因，而不是泛化报错

**为什么重要：**
用户最害怕“误操作”和“状态不确定”，明确反馈会显著提升安全感。

---

### 18. 长耗时任务缺少过程感
**当前问题：** 导入、联网补全、批量扫描、未来的 BibTeX/RIS 导入都可能耗时，但用户很难判断当前进度。

**建议补充：**
- 顶部或底部任务中心：显示正在导入 / 正在识别元数据 / 正在导出
- 批量任务进度条、剩余数量、失败数量
- 支持取消长任务或至少支持后台继续
- 批量任务完成后给出汇总结果（成功 X、失败 Y、跳过 Z）

**为什么重要：**
“等待但不知道发生了什么”是最典型的桌面软件挫败点之一。

---

### 19. 阅读中断恢复能力不足
**当前问题：** 用户离开应用再回来时，缺少“继续上次工作”的自然承接。

**建议补充：**
- 自动记忆每篇 PDF 的阅读位置、缩放比例、侧边栏状态
- 重启后恢复上次打开的标签页和当前活跃文献
- 提供“最近打开”视图
- 对未完成编辑内容（笔记、元数据）给出自动保存与恢复

**为什么重要：**
研究工作往往是中断式进行的，恢复现场的速度直接影响黏性。

---

### 20. 批注与笔记缺少“收束”动作
**当前问题：** 用户可以做批注、写笔记，但读完论文后缺少一个自然的整理闭环。

**建议补充：**
- “导出本篇阅读摘要”动作：把批注、笔记、引用整理成一份 Markdown
- 批注快速加入笔记时支持模板化插入（页码、摘录、评论）
- 提供“本篇已完成阅读 / 待复习 / 重点论文”等轻状态
- 支持为论文添加星标、阅读状态、最后阅读时间

**为什么重要：**
用户真正需要的不是“能标”，而是“读完之后能回顾、能整理、能复用”。

---

### 21. 拖拽与批量操作的可预期性仍需加强
**当前问题：** 即使功能存在，如果用户在操作前看不到清晰提示，也会觉得不敢用。

**建议补充：**
- 拖到目标文件夹时显示明确高亮、文案提示或投放线
- 批量选择后显示浮动操作条（移动、打标签、导出、删除）
- 列表项 hover 态提供更明确的可操作暗示
- 对不可执行操作给出禁用态和原因说明

**为什么重要：**
桌面端高频操作强调“可预判”，让用户在操作前就知道系统会怎么响应。

---

### 22. 设置项缺少“解释成本优化”
**当前问题：** 设置功能增加后，如果只有选项没有解释，会让用户不知道改了会发生什么。

**建议补充：**
- 每个设置项增加简短说明文字
- 涉及行为变化的设置提供即时预览
- 恢复默认设置入口
- 对高影响选项给出推荐值或默认理由

**为什么重要：**
好的设置不是“参数多”，而是让用户能放心地改。

---

### 23. 可访问性与视觉舒适度仍未系统化
**当前问题：** 目前已有深色模式基础，但还缺少系统性的无障碍与视觉舒适设计。

**建议补充：**
- 更完整的键盘可达性：Tab 焦点顺序、Esc 关闭、Enter 确认
- 焦点态高亮统一设计
- 支持更大的界面字号 / 更高对比度模式
- 给图标按钮补全 tooltip 与 aria 标签
- 降低一次界面信息密度，避免 MetaPanel 过于拥挤

**为什么重要：**
“看得清、点得到、不会累”是长期使用软件的基础，不只是无障碍议题。

---

### 24. 缺少建立“信任感”的细节
**当前问题：** 学术用户非常在意数据安全、文件位置、是否会误覆盖本地资料。

**建议补充：**
- 明确展示附件真实文件路径与当前归档位置
- 在删除 / 移动 / 重命名时说明“是否影响原文件”
- 提供自动备份 / 手动导出数据库的入口
